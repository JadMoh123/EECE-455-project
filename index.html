Section 02 / The Substitution Box

The heart of confusion.

SubBytes is AES's only non-linear step. Every byte is replaced by looking it up in a 256-entry table. The math behind it is elegant, multiplicative inverse in GF(2⁸) followed by an affine transformation, but in software it's just one array lookup. Hover any cell to inspect a byte substitution. Section 03 / Finite Field Arithmetic

The math behind MixColumns.

AES lives in GF(2⁸), the Galois field of 256 elements. Bytes are polynomials. Addition is XOR. Multiplication uses the irreducible polynomial x⁸ + x⁴ + x³ + x + 1 (0x11B). Type two bytes below and watch the shift-and-XOR algorithm compute their product.

Step-by-step

The algorithm processes one bit of B per step. If the bit is set, A is added (XORed) to the running total. Then A is shifted left and, if it overflows 8 bits, reduced by XORing with 0x1B.

Section 04 / Diffusion & the Avalanche Effect

Flip one bit. Watch chaos unfold.

AES is designed so that a one-bit change in the input cascades through the cipher, affecting roughly half of all output bits within a few rounds. This is the avalanche effect, the hallmark of a strong block cipher. Type two slightly different plaintexts and watch them diverge.
